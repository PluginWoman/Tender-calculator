# Дизайн: переработка блока «Идентификация закупки»

**Дата:** 2026-07-07  
**Файл:** `src/components/calculator/CalculatorForm.tsx`

---

## Цель

Разделить информацию о закупке и параметры расчёта на две логические группы внутри первого блока, чтобы пользователю было проще ориентироваться при заполнении формы.

---

## Что меняется

Только JSX блока 0 (`/* Block 0: Identification */`) и удаление блока 5 (`/* Block 5: Profit */`). Стейт, хук, логика расчётов — без изменений.

---

## Перемещения полей

| Поле | Было | Стало |
|------|------|-------|
| Название закупки | Блок 0, ряд 1, кол. 1 | Группа «Закупка», ряд 1, кол. 1 |
| Номер в ЕИС | Блок 0, ряд 1, кол. 2 | Группа «Закупка», ряд 1, кол. 2 |
| Тип предмета закупки | Блок 0, ряд 2, кол. 1 | Группа «Закупка», ряд 2, кол. 1 |
| Категория (для нормативов) | Блок 0, ряд 2, кол. 3 | Группа «Закупка», ряд 2, кол. 2 |
| НМЦК заказчика | Блок 0, ряд 3, кол. 1 | Группа «Параметры расчёта», ряд 1, кол. 1 |
| Целевая рентабельность (%) | Блок 5 «Целевая прибыль», ряд 1, кол. 1 | Группа «Параметры расчёта», ряд 1, кол. 2 |
| Система налогообложения | Блок 0, ряд 2, кол. 2 | Группа «Параметры расчёта», ряд 2, кол. 1 |

---

## Что удаляется

- Старый третий ряд в блоке 0 (НМЦК + два пустых `<div />`)
- Весь `<FormBlock title="Целевая прибыль">` (блок 5) вместе с содержимым

---

## Что не меняется

- `onChange` логика всех полей — без изменений
- Подпись «Рекомендуется 10–20%» сохраняется под полем рентабельности
- Поле «Сумма прибыли» в форме не отображается (это результат расчёта, не поле ввода)
- CSS классы `costGroupsContainer` и `blockContent` уже существуют — переиспользуются
- Все остальные блоки формы — без изменений
