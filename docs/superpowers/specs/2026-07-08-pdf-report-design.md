# PDF-отчёт калькулятора тендера

**Дата:** 2026-07-08  
**Статус:** Approved

---

## Цель

Добавить кнопку «Скачать отчёт PDF» в боковую панель результатов. По клику генерируется и скачивается трёхстраничный PDF с итогами расчёта, детализацией затрат и маркетинговой страницей.

---

## Технологии

- **PDF-генерация:** `@react-pdf/renderer` — декларативные React-компоненты, рендер в браузере, настоящий вектор (текст выделяется, шрифты встроены).
- **QR-код:** статичный PNG-файл `src/assets/qr-zakupki.png`, сгенерированный один раз для URL `https://zakupki.tochka.com/search/`.
- **Диаграмма:** SVG-примитивы `@react-pdf/renderer` (`<Svg>`, `<Circle>`, `<G>`) — кольцевая диаграмма без внешних чарт-библиотек.

## Кнопка запуска

В `ResultsSidebar`, под блоком итоговых цифр, рендерится только когда `results.isReady === true`:

```tsx
import { PageAction } from '@pluginwoman/t-ds'
import { ArrowDownIncomingCircle } from '@pluginwoman/t-ds/icons'

{results.isReady && (
  <PageAction
    title="Скачать отчёт PDF"
    leftAccessory={<ArrowDownIncomingCircle />}
    onClick={() => generatePdf(state, results)}
  />
)}
```

`generatePdf` — async-функция, во время генерации блокировать кнопку не требуется (генерация быстрая, < 1 сек).

## Имя скачиваемого файла

```
Расчёт_тендера_<slug>_<дата>.pdf
```

`<slug>` — первое непустое из: название закупки → номер ЕИС → «без_названия».  
Спецсимволы и пробелы заменяются на `_`, длина slug ограничена 40 символами.  
`<дата>` — `state.calcDate` в формате `YYYY-MM-DD`.

Пример: `Расчёт_тендера_Поставка_оргтехники_2026-07-08.pdf`

## Страница 1 — Сводка

### Шапка
- Слева: заголовок «Расчёт рентабельности тендера» (крупный)
- Справа: дата расчёта (`state.calcDate`)

### Блок идентификации
| Поле | Значение |
|---|---|
| Название закупки | `state.purchaseName` (или «—» если пусто) |
| Номер в ЕИС | `state.purchaseNumber` (или «—» если пусто) |

### Таблица итогов
| Параметр | Значение |
|---|---|
| НМЦК | `state.nmcc` |
| Итоговая цена | `results.finalPrice` |
| Себестоимость | `results.fullCost` |
| Прибыль | `results.profitAmount` |
| Налоги | `results.taxAmount` |

### Блок решения
Цветной фон по `results.verdict`:
- `success` → зелёный фон
- `caution` → жёлтый фон
- `danger` → красный фон

Содержимое:
- Иконка + заголовок: «Участвовать» / «Осторожно» / «Отказаться»
- Запас/дефицит: `results.nmccDiff` в ₽ и `results.nmccDiffPercent` в %
- Пояснение: одна строка текста по вердикту (аналогично тому, что показывает `ResultsSidebar`)

### Диаграмма структуры цены
Кольцевая SVG-диаграмма + легенда справа.

**Сегменты** (5 штук):
| Статья | Значение | Цвет |
|---|---|---|
| Прямые затраты | `results.directCosts` | `#7c3aed` |
| Накладные | `results.overheadTotal` | `#a78bfa` |
| Специфические | `results.specificCosts` | `#c4b5fd` |
| Прибыль | `results.profitAmount` | `#10b981` |
| Налоги | `results.taxAmount` | `#6ee7b7` |

В центре кольца — `results.finalPrice` (итоговая цена).

**Легенда** (справа от кольца): для каждой статьи — цветная точка, название, сумма в ₽ жирным, процент от `finalPrice` под ней.

Доли считаются как `value / finalPrice * 100`, округляются до целых. Если сумма долей ≠ 100% из-за округления — корректируем самую большую статью.

## Страница 2 — Детализация затрат

### Исходные параметры
| Параметр | Значение |
|---|---|
| Тип предмета | «Товар (Поставка)» / «Услуга» |
| Категория | человекочитаемое название из `CATEGORY_OPTIONS` |
| Система налогообложения | «ОСНО (НДС 20%)» / «УСН Доходы 6%» / «УСН Доходы-Расходы 15%» |
| Целевая рентабельность | `state.profitPercent %` |
| Коэффициент накладных | `state.overheadPercent %` |

### Прямые затраты

**Если `purchaseType === 'service'`:**

Таблица ФОТ:
| Специалист | Кол-во | Часы | Ставка | Стоимость |
|---|---|---|---|---|
| … | … | … | … ₽/ч | … ₽ |

Строка «Итого ФОТ» = сумма `qty × hours × rate` по всем специалистам.  
Строка «Страховые взносы (30%)» = ФОТ × 0.30.

**Если `purchaseType === 'goods'`:**

Таблица товаров:
| Наименование | Кол-во | Цена | Таможня | Логистика | Упаковка | Итого |
|---|---|---|---|---|---|---|
| … | … | … ₽ | … ₽ | … ₽ | … ₽ | … ₽ |

Строка «Итого по товарам».

### Прочие затраты

Активные статьи определяются в `generatePdf` по значению в state — без передачи `activeArticles`:

```ts
const activeArticleItems = ARTICLE_CONFIG.filter(a => (state[a.stateKey] as number) > 0)
```

Если таких нет — секция «Прочие затраты» не рендерится.

Каждая статья: название (из `ARTICLE_CONFIG[].label`) + сумма.

### Накладные расходы

Три строки:
- База расчёта: «ФОТ» (для IT, медицина, прочие) или «Прямые затраты» (для товары, строительство)
- Коэффициент: `state.overheadPercent %`
- Итого: `results.overheadTotal` в ₽

### Специфические затраты

Показываются только поля со значением > 0. Метки и поля:

| Проверяемое поле | Метка в PDF |
|---|---|
| `bidSecurityAmount` | Обеспечение заявки |
| `bidSecurityDays` | Срок заморозки (дней) |
| `contractSecurityAmount` | Обеспечение контракта |
| `contractDurationDays` | Срок контракта (дней) |
| `paymentDelayDays` | Отсрочка платежа (дней) |
| `riskReservePercent` | Резерв рисков |
| `tenderMgmt` | Тендерный менеджмент |
| `warrantyReserve` | Гарантийный резерв |
| `bidSecCost` (из results) | Стоимость заморозки заявки |
| `contSecCost` (из results) | Стоимость банковской гарантии |
| `delayCost` (из results) | Стоимость отсрочки |
| `riskAmount` (из results) | Сумма резерва рисков |

Если ни одно поле не заполнено — секция не рендерится.

## Форматирование чисел в PDF

- Денежные значения: `Intl.NumberFormat('ru-RU', { style: 'currency', currency: 'RUB', maximumFractionDigits: 0 })` — через существующий `fmtMoney` из `src/utils/format.ts`
- Проценты: `n.toFixed(1) + ' %'` (одна цифра после запятой)
- Дата: отображается как есть из `state.calcDate` (ISO: YYYY-MM-DD)

---

## Обработка крайних случаев

- Название и номер ЕИС пустые → показываем «—»
- Таблица специалистов пустая (теоретически невозможно, но) → не рендерим секцию ФОТ
- Специфические затраты все нулевые → не рендерим секцию
- `verdict === null` → блок решения не рендерится (но `isReady` требует `nmcc > 0`, так что verdict всегда есть при `isReady`)

## Зависимости

```bash
npm install @react-pdf/renderer
```

Добавить типы (если нужны отдельно):
```bash
npm install --save-dev @types/react-pdf
```

`@react-pdf/renderer` включает собственные типы — `@types/react-pdf` обычно не нужен.
